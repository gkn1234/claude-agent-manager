# Coding Conventions

This document provides a high-level summary of the project's coding conventions, derived from configuration files and observed source code patterns.

## 1. Core Summary

This is a Next.js 16 App Router project using TypeScript in strict mode, pnpm as the package manager, Tailwind CSS v4 for styling, shadcn/ui (new-york style) with Radix UI primitives for components, Drizzle ORM with SQLite for persistence, and Zod for validation. The UI language is Chinese (zh-CN).

## 2. Source of Truth

- **Package Manager:** `package.json` (pnpm section) - pnpm is the only supported package manager. Never use npm or yarn.
- **TypeScript Config:** `tsconfig.json` - Strict mode enabled, target ES2017, path alias `@/*` maps to `./src/*`.
- **ESLint Config:** `eslint.config.mjs` - Uses `eslint-config-next` with core-web-vitals and TypeScript presets.
- **PostCSS Config:** `postcss.config.mjs` - Uses `@tailwindcss/postcss` (Tailwind CSS v4).
- **shadcn/ui Config:** `components.json` - Style: `new-york`, RSC enabled, icon library: `lucide`, CSS variables enabled with `neutral` base color.
- **Next.js Config:** `next.config.ts` - Default configuration, no custom settings.

## 3. Key Conventions

### Framework Patterns

- **App Router:** All pages live under `src/app/`. Server Components by default; add `'use client'` directive only when client interactivity is needed.
- **API Routes:** Use Next.js Route Handlers (`route.ts`) exporting named HTTP method functions (`GET`, `POST`, `PUT`, `DELETE`). Return `NextResponse.json()`.
- **Layouts:** Root layout at `src/app/layout.tsx` wraps all pages with `AppShell`.

### Component Patterns

- **UI Primitives:** Located in `src/components/ui/`, generated by shadcn/ui CLI. Use `cva` (class-variance-authority) for variant styling.
- **Composition:** Components use `cn()` from `src/lib/utils.ts` (`clsx` + `tailwind-merge`) for conditional class merging.
- **Radix UI:** Imported as `radix-ui` (unified package, e.g., `import { Slot } from "radix-ui"`).
- **Feature Components:** Organized by domain under `src/components/{domain}/` (e.g., `projects/`, `commands/`, `nav/`).
- **Props:** Use TypeScript interfaces, not `type` aliases, for component props. Extend `React.ComponentProps<>` for HTML element props.

### Code Style

- **Exports:** Named function components (not arrow functions) exported by name. No default exports for components; pages use `export default function`.
- **Hooks:** Custom hooks in `src/hooks/` prefixed with `use-` (kebab-case files). Use `'use client'` directive.
- **Imports:** Use `@/` path alias exclusively for project imports. Group: external libs, then `@/` imports.
- **Data Layer:** Drizzle ORM schema in `src/lib/schema.ts`, database singleton in `src/lib/db.ts`. UUID primary keys as text.
- **Validation:** Zod for request body validation in API routes.
- **No semicolons** in UI component files (shadcn convention); **semicolons used** in API routes and lib files.
