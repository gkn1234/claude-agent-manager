## Claude Code 远程任务派发系统设计

我希望开发一个 Web 应用，支持远程、手机端向部署在 ECS 上的 Claude Code 派发任务，并能够实时监控任务状态

应用交互移动端优先(毕竟远程派发，手机会高频使用，务必保证手机端使用体验)。

思路参考：/Users/macbookair/Desktop/projects/claude-agent-manager/docs/2026-02-27-claude-code-task-dispatch-system-design.md

## 核心原则

所有的原子操作都不能耦合 UI，必须可以通过 API 接口完成，这样才方便接入 LLM 进行驱动。

## 概念

### 项目

基于 Git 仓，用于划分工作空间，用户创建的所有任务、会话必须基于项目。

#### 创建项目

用户创建项目的时候，必须基于一个 Git 仓。

Git 仓可以通过下面方式获取：
1. 用户选取已有本地项目（已经创建或者 clone 到本地的项目）
2. 用户输入远程 git 地址，clone 代码后创建本地项目
3. 用户输入项目名称，在本地新建目录，建立新项目

#### 删除项目
删除项目是危险操作，将从本地直接删除整个工作目录.

#### 项目信息
展示：
- 本地工作目录
- 远程 Git 仓（可选）
- 任务列表

### 任务
在项目中规划一项工作

#### 创建任务
流程：

任务初始化：
- 用户在项目中新建任务，输入任务描述
- 在项目工作目录发起【初始化指令】，指导 Claude Code 基于 git worktree 创建任务专用的工作空间，并创建初始的 PROGRESS.md，记录进度
- 工作空间强制要求为：<项目工作目录>/.worktrees/<任务工作目录>，并注意在 .gitignore 中添加。可以使用 superpowers:using-git-worktree
- Claude Code 可能会认为原始描述过于庞大，建议分解为多个任务。可以通过创建任务 API 发起多个其他任务

初始化完成：
- 之后可以在任务中发起正式工作指令

#### 删除任务
将直接删除对应的 worktree 以及任务工作目录

#### 任务信息
展示：
- 分支名称
- 任务工作目录
- 指令列表

### 指令
推进任务的每一轮与 AI 的交互

#### 发起指令
用户选择任务后，可以发起会话，派发指令

当前有指令进行中的任务，不可派发新的指令，必须中止或等待现有指令完成

派发指令，默认推入执行列表，变为 "排队中" 或 "进行中" 状态(取决于是否有空闲资源启动新的 claude code)。

当然用户也可以选择不立即执行(默认立即执行)，指令将是"未开始"状态。

未开始的指令也可以手动激活

#### 指令信息
- 状态：未开始、排队中、进行中、完成、失败、中止
- 类型：规划（Plan mode）/执行
- 内容：用户发送给 Claude Code 的 prompt
- 结果：Claude Code 返回输出的结果，需要用 markdown 展示

#### 指令执行
系统会检查当前有多少个 Claude 示例运行，剩余资源是否能启动新的 Claude。

当资源足够时，系统会自动发起 Claude 命令，从指令列表里选取优先级最高的一条执行

用户可以在 UI 界面通过拖拽调整指令优先级，当然，也可以让 claude code 通过 API 完成

指令执行时，会读取任务工作目录，在正确的地方执行，resume 会话历史(统一任务下的历史交互)，并实时记录 PROGRESS.md

一轮交互完成，获取 claude 输出，视为【完成】

发生错误，结果展示错误信息，视为【失败】

可以手动中止，系统会杀掉进程，状态变为【中止】

#### 指令管理

指令管理的集中方式

##### 系统首页
系统首页会以类似 Github Project 的阶段分组的版式，展示所有指令，这里可以拖拽调整指令优先级

可以添加项目、任务过滤器，

但是过滤后，不可调整优先级

过滤视图下，可以通过卡片进入任务视图

##### 任务视图

可以看到一路下来的历史指令，可以在不忙碌的任务下派发新指令
